Github Wikiのコンテンツのバックアップ方法

Github WikiはGitで管理されているので、WikiのGitリポジトリを「git clone」するだけです。ページ全体をバックアップできます。 

git clone https://github.com/yuki-kimoto/perlzemiubuntu.wiki.git

ローカルで編集してpushする

GithubのWikiは単なるGitリポジトリなので、ローカルで編集して「git push」するということも可能です。

全体的に修正したい内容がある場合に、便利です。

Gitは、分散リポジトリ型のソースコード管理システムです。
リモートとローカルで完全に同一のソースコード管理情報を保有できるため、バックアップが簡単。
ブランチの作成が高速。完全な履歴を保有しているので、ソースコードをいつでも過去の状態に戻すことができます。
PerlによるWebシステム開発、データ分析、ディープラーニングを使ったAIのソースコード管理に活用できます。
GitはUnix/Linuxミドルウェアのひとつで、主にC言語で書かれ、部分的にPerlで書かれています。
