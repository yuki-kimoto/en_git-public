GithubのWikiの編集

GithubのWikiを編集してみましょう。右上の「Edit」をクリックします。

Wikiのコンテンツの編集方法や、保存方法は、GithubのWikiの新規作成で解説したものと同じです。

Wikiの書き方 - マークダウン記法の基礎

Wikiを書く場合にマークダウン記法を使いますが、書き方をいくつか覚えておきましょう。

見出し

見出しは以下です。 

# 見出し1
## 見出し2
### 見出し3

コード

コード。 

```
use strict;
use warnings;
```

リスト

リスト。 

* 項目1
* 項目2
* 項目3

テーブル

テーブル。 

|  TH  |  TH  |
| ---- | ---- |
|  TD  |  TD  |
|  TD  |  TD  |

リンク

ページリンク。Wikiに存在しないページ名の場合は、リンクが赤くなり、ページの新規作成ページにリンクされます。 

[[ページ名]]

ページ名と異なるタイトルをつける場合。 

[[環境構築手順書|20200618102400]]

外部リンク。 

[[木本システム|https://kimoto-system.co.jp/]]

Gitは、分散リポジトリ型のソースコード管理システムです。
リモートとローカルで完全に同一のソースコード管理情報を保有できるため、バックアップが簡単。
ブランチの作成が高速。完全な履歴を保有しているので、ソースコードをいつでも過去の状態に戻すことができます。
PerlによるWebシステム開発、データ分析、ディープラーニングを使ったAIのソースコード管理に活用できます。
GitはUnix/Linuxミドルウェアのひとつで、主にC言語で書かれ、部分的にPerlで書かれています。
